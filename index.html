Unidad 1: Elementos de un sistema de bases de datos

Mapa mental interactivo basado en los temas de la unidad. Todo el contenido está explicado con palabras simples.

  • Una base de datos es una forma ordenada de guardar información para consultarla, actualizarla y analizarla con rapidez.
    • Hoy las empresas necesitan responder preguntas complejas en poco tiempo. Por eso, diseñar bien una base de datos desde el principio ayuda a que el sistema sea rápido y se adapte a cambios.
    • Reconocer las piezas clave de un sistema de bases de datos y entender cómo apoyan la gestión de información en una organización.
    • Es un repositorio organizado: los datos no quedan sueltos, sino estructurados para poder encontrarlos fácilmente y tomar decisiones con menos errores.
      • Consultas rápidas, posibilidad de crecer sin rehacer todo, mejor organización y más control sobre la información.
        • Independencia: los datos pueden usarse desde distintos programas.
        • Menos repetición: se evita duplicar información innecesaria.
        • Seguridad: se controla quién puede ver o modificar cada dato.
      • Separar en tablas para evitar repeticiones, asegurar datos correctos y mantener integridad para que reportes y análisis sean confiables.
    • Un DBMS es el software que define cómo se guardan los datos y permite consultarlos o cambiarlos de forma controlada.
      • Definir estructura (tablas, campos, reglas) y manipular información (insertar, consultar, actualizar, eliminar).
      • Personas, equipos, programas y datos trabajando juntos.
        • Física: dónde y cómo se guarda realmente la información.
        • Conceptual: representación lógica del negocio.
        • Vista: parte de la información que ve cada usuario según permisos.
        • Relacional: tablas conectadas por campos comunes.
        • Red: registros unidos por múltiples enlaces.
        • Jerárquico: estructura tipo árbol (padre-hijo).
      • Incluye componentes para definir estructuras (DDL), manipular datos (DML), procesar consultas y traducir peticiones entre distintos niveles.
    • Una base de datos útil no solo guarda datos: también protege su exactitud y coherencia.
      • Significa que los datos son válidos y consistentes. Se aplican reglas para evitar errores, por ejemplo impedir un teléfono en un campo de fecha.
      • Es repetir la misma información en varios lugares. Esto gasta espacio y puede generar contradicciones.
      • Todas las copias y referencias de los datos deben coincidir. Si se actualiza un dato, el cambio debe reflejarse en todo el sistema.
    • Son los “objetos” sobre los que guardamos información: personas, productos, eventos, lugares o ideas del negocio.
        • Fuerte: existe por sí sola.
        • Débil: depende de otra entidad para existir.
      • Grupo de elementos con características comunes, por ejemplo ciudades de un país o jugadores de un torneo.
    • Los campos son columnas de una tabla. Los atributos son las propiedades que describen cada entidad.
      • Sirven para estructurar la tabla y también para conectar tablas entre sí mediante identificadores comunes.
      • Cada atributo toma valores dentro de un dominio (conjunto permitido de valores), por ejemplo fechas válidas o códigos definidos.
        • Simple o compuesto.
        • De valor único o multivaluado.
        • Derivado (se calcula a partir de otros datos).
    • Una relación conecta entidades y expresa cómo dependen entre sí dentro del sistema.
        • Reflexiva: una entidad se relaciona consigo misma.
        • Binaria: participan dos entidades.
        • N-aria: participan tres o más entidades.
      • Indica cuántas ocurrencias de una entidad pueden asociarse con otra. Ayuda a evitar duplicados y errores en reportes.
      • Cada registro de A se relaciona con uno de B, y cada registro de B con uno de A.
      • Un registro de A puede tener varios en B, pero cada registro de B pertenece a un solo A.
      • Varios registros de A se asocian con varios de B. Suele requerir una tabla intermedia para implementarse.
      • MySQL Workbench: diseño y administración visual de bases de datos MySQL.
      • Diagrams.net: creación de diagramas (flujo, UML, redes, etc.).
      • XAMPP: entorno local con Apache, MySQL/MariaDB, PHP y Perl.
    • Idea central: diseñar bien una base de datos permite almacenar información confiable, relacionarla correctamente y consultarla de forma eficiente.
